# 🎵 音乐 URL 自动获取工具 - 使用指南

## 📋 功能说明

该脚本可以自动：
1. ✅ 读取 `js/music-data.js` 中的所有 105 首歌曲
2. ✅ 从免费音乐 API（Deezer）查询每首歌曲的 URL
3. ✅ 自动为每首歌添加 `url` 字段
4. ✅ 保存到文件，立即可用

---

## 🚀 快速开始

### **简单方法（推荐 Windows 用户）**

**直接双击运行：**
```
run-fetch-urls.bat
```

脚本会自动检查、下载并更新所有 URL。

---

### **手动方法（所有系统）**

1. **打开终端/命令行**
   - Windows: 打开 PowerShell 或 CMD
   - Mac/Linux: 打开 Terminal

2. **导航到项目目录**
   ```bash
   cd d:\Visual Studio Code\projects\简版网页
   ```

3. **运行脚本**
   ```bash
   node fetch-music-urls.js
   ```

4. **等待完成**
   - 脚本会显示进度条
   - 预计耗时 2-5 分钟（取决于网络速度）

## ⚠️ 重要说明

### **API 限制**
- Deezer API 有访问频率限制，如果请求过快可能失败
- 脚本会自动使用演示音乐作为备选方案
- 某些歌曲可能无法在 Deezer 中找到

### **URL 特性**
- ✅ 可直接播放
- ⚠️ 部分为 30 秒预览版本
- ✅ 支持所有现代浏览器
- ✨ 无需登录或 API Key

### **如果 URL 无法播放**

可能的原因和解决方案：

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 无声音 | 浏览器跨域限制 | 使用本地 MP3 文件 |
| 播放错误 | URL 过期 | 重新运行脚本 |
| 获取失败 | API 不可用 | 检查网络连接 |

## 📞 常见问题

### Q: 脚本出错怎么办？
**A:** 检查：
1. Node.js 是否正确安装（运行 `node --version`）
2. 网络连接是否正常
3. music-data.js 文件是否完整

### Q: URL 是否会过期？
**A:** 可能会。如果播放失败，重新运行脚本即可刷新。

### Q: 能否使用其他音乐源？
**A:** 可以修改脚本中的 `MUSIC_SOURCES` 配置，添加其他 API。

### Q: 为什么某些歌曲没有找到？
**A:** API 可能没有该歌曲，脚本会自动使用演示音乐替代。
